FAQ

Q: What are the supported platforms?

Xel works only in environments that implement the latest Web Platform APIs such as Custom Elements, Shadow DOM, Web Animations, SVG 2, Pointer Events, CSS variables, Constructable Stylesheets and ES2020.

The supported environments are as follows:

  • Chrome >= 85
  • Edge >= 85
  • Vivaldi >= 3.3
  • Opera >= 71
  • Brave >= 1.13
  • Electron >= 10.0
  • NW.js >= 0.48
  • Android WebView >= 85
  • Microsoft Edge WebView2 >= 85
  • Qt WebEngine >= 5.15

The following environments are not supported:

  • Firefox
  • Safari
  • WebKit-based browsers
  • WebKit WebView

Q: Why is my app broken after updating Xel?

Xel 0.9 (released in 2021) introduced major API changes that broke backward compatability with earlier alpha versions (0.0.x).

You should carefully review the new setup guide and the documentation for each element and update your project accordingly.

Q: Can I use Xel with React, Vue, Angular, Polymer or some other framework?

Xel is framework-agnostic. Any framework that supports custom elements should be compatible with Xel.

Custom elements provided by Xel have names starting with x- prefix. You should ensure that your app and any other frameworks used by your app do not use the same prefix to avoid naming clashes.

Q: Can I use Xel to build a commercial app with it?

Xel JavaScript code is licensed under the MIT License and you can use it freely in commercial projects.

The Cupertino themes which implement macOS Human Interface Guidelines have a more restrictive license. You are allowed to use them exclusively with macOS operating system. Please check the full license text for details.

Q: Is Xel used by any large projects or organizations?

Xel was originally built for and is used extensively by Boxy SVG editor.

The toolkit is also used by several smaller projects such as Type Fu touch typing tutor.

Q: How do I prevent a flash of unstyled content during the initial page load?

Since Xel themes are loaded asynchronously, you can't just rely on the document load event. Instead you have to wait for the Xel.whenThemeReady promise to resolve:

import Xel from "/node-modules/xel/xel.js"; document.body.hidden = true; document.addEventListener("load", async () => { await Xel.whenThemeReady; document.body.hidden = false; });
